# Whole-app design critique — EasySynQ QMS

App Design Health: 26/40 (Good). AI-slop: mostly-human. Method: code-grounded (auth-gated live app) + detector/grep Assessment B.

## Heuristics (app-wide)
- Visibility of System Status: 3/4 — State is triple-encoded (label+glyph+colour) on the core badges nearly everywhere — genuinely strong. Dragged down app-wide by two systemic gaps: no freshness/'as of' timestamp on any status board (Home, Objectives, Compliance, Drift) in a product whose whole thesis is proving currency, and the TasksInbox/MyTasksRail name no subject so they read as a count, not a triage surface. Loading vocabulary is also split (bare Loader vs skeleton).
- Match System & Real World: 2/4 — Vocabulary is ISO-true where humanised (clause spine, PDCA, INPUT/OUTPUT labels, 'governing revision'), but raw backend enums and machine keys leak to the most-scrutinised readers across at least five surfaces: CAPA close_state ('RootCause','ActionPlan'), DCR impact dimensions ('effectivity_transition') + change_significance, MR SummaryTable (k.replace(/_/g,' ') → 'verify failed at <ts>'), drift counts (snake_case in mono), inbox stage_key 'action:<uuid>'. A schema leaking into the UI is the legacy-QMS anti-reference.
- User Control & Freedom: 3/4 — Drawers/palettes/modals get Escape+scrim+cancel for free and URL-as-state makes views shareable/back-safe. But irreversible compliance acts fire on a single click with no confirm and no undo across the app: NCR disposition (ISO 8.7, one-shot), document Release, MR Submit/Release/Close + Remove-output, audit Issue-report/Close, CAPA Return-to-root-cause, ingestion bulk-Exclude. Mobile nav also never closes on navigation.
- Consistency & Standards: 2/4 — The app's defining weakness, visible only across surfaces. TWO non-interoperable colour conventions render different hues for the same state — CSS-var badges (StateBadge family) vs Mantine palette names (objectives RAG, home rag.ts, audits, CAPA severity, MR close_state) — so token-green and Mantine-green sit in the SAME row, and badge contrast leans on Mantine light-variant math over raw hues rather than the AA-tuned -soft/-text token pairs. ~12 near-identical badge META maps each redefine their own glyph (amber = ▲ here / ◔ there; the Objectives register RAG badge has no glyph at all). Emoji icons (TopBar 🔍⌖🔔👤; ingestion 📄🔒🛡) clash with the geometric set. Three names for one CAPA lifecycle on one screen.
- Error Prevention: 3/4 — Genuinely strong and consistent: capability-gating prevents show-then-403, comment-required gates, sign-checkbox gates, per-mount idempotency keys, server-authoritative close gates surfaced honestly, the ingestion included_candidate guard makes a bad bulk-sweep structurally impossible, unit-locked measurement entry. Held back only by the missing confirm-on-irreversible (heuristic 3) and free-text numeric fields in the Objectives modals.
- Recognition vs Recall: 2/4 — Glyph+label badges support recognition, but recall is forced in the surfaces users live in most: the general TasksInbox and Home rail name no document/subject; DCR and MR registers hide the target identity ('Document'/'—'); MergeMenu and DCR impact identify rows by raw UUID; the signing checkbox attests without recapping what is being signed. The inconsistent glyph language itself must be re-learned per feature, and is undocumented in-product.
- Flexibility & Efficiency: 2/4 — Systemic gap for Mara/Ken (the daily power users): almost no register has text search or column sort, no bulk actions in the general task queue or CAPA/audits/objectives, no keyboard row-nav, no saved/persisted filter views, no density toggle at the system level. The ⌘K palette is search-only (not a true command palette). Genuine bright spots — Library density toggle, redline n/p nav, ingestion bulk bar + palette keyboard nav — prove the team can do this; it just isn't applied evenly.
- Aesthetic & Minimalist: 4/4 — The app's strongest dimension and held with real discipline: one indigo accent, layered neutral surfaces, 8px scale, system-font stack, no gradients/eyebrows/side-stripes/hero-metric templates anywhere. Calm, audit-grade tone throughout. Only blemishes are the emoji glyphs and a few raw-key/JSON.stringify debug-grade dumps (MR SummaryTable, drift counts, CAPA ContentBlock).
- Error Recovery: 3/4 — Calm, specific, plain-language recovery copy is consistent and on-brand (403→quiet no-access, 404→back-link, lock-conflict→force-unlock, SoD→human message, server-authoritative 409s). But the copy is hand-rolled per page (string-duplicated) and almost no error Alert offers a Retry button — the user is told to 'try again' with no button and must reload the SPA. No shared ErrorState/RetryBoundary primitive. useAckCount returns 0 on error (a silent false all-clear in a compliance tool).
- Help & Documentation: 2/4 — Per-control microcopy is genuinely good (lifecycle button explainers, the N9 'not a compliance verdict' note, forbidden panels that name the exact permission + roles). But there is no system-level help layer: no legend/tooltip for the glyph vocabulary an auditor meets cold, no glossary for domain terms (effectivity_transition, disposition tokens, the 7/9 lifecycle states), and no first-run onboarding anywhere in the shell — Sam and Olsen meet the QMS with no orientation.

## AI-slop verdict
Mostly human — and confidently so. Across all 11 surfaces the absolute AI-slop bans are absent: no side-stripe accent borders, no gradient/clip text, no glassmorphism, no hero-metric template, no uppercase tracked eyebrows used as scaffolding, no 01/02/03 numbered markers, no identical icon-heading-text card grid, no decorative motion. The code reads as a real domain engineer's work: dense load-bearing comments that cite ISO clauses, SoD invariants, 409 codes and Codex fix references; status triple-encoded by intent not template; error-prevention logic that encodes actual backend contracts. The tells that exist are human-process smells, not generation smells — the kind that accrete when each slice ships in isolation without a unifying design pass: ~12 copy-pasted badge META maps that drift, two parallel colour conventions, raw enum/key dumps shipped where a designer-in-the-loop would have caught them, and decorative emoji (the single clearest consumer-SaaS tell, and an explicit PRODUCT.md anti-reference). A Linear/Stripe-fluent user would trust this app and pause at exactly three things: the emoji, the mismatched greens in one row, and the raw 'RootCause'/'effectivity_transition' tokens.

## Overall impression
EasySynQ is a genuinely calm, audit-grade QMS that already delivers its hardest brand promise — restraint — with rare discipline, and gets the load-bearing safety mechanics (capability-gating, server-authoritative close gates, idempotency, colour-safe core badges, honest permission degradation) right almost everywhere. What it does NOT yet do is read as one authored hand. The same product, built slice-by-slice, has accreted two colour conventions (token-green ≠ Mantine-green in the same table row), ~12 drifting badge components with an inconsistent glyph language, raw backend enums/keys leaking to the exact readers — the external auditor and the Quality Manager — whose trust the product is built on, and a near-total absence of the power-user efficiency (search/sort/bulk/keyboard) that Mara needs daily. The single biggest opportunity is a focused systematisation pass: extract ONE StatusBadge primitive over ONE canonical glyph+token map, register the --es functional hues as Mantine virtual colours so a single colour system wins, and add three shared primitives (PageLoading, ErrorState-with-retry, ConfirmDestructive). That one effort closes the top cross-surface issue (consistency), the recovery gap, AND the irreversible-action gap at once — converting a strong-but-uneven app into a coherent one.

## What is working
- Colour-safe status is correct on the core badge family across every surface — StateBadge / TaskStateBadge / ReviewStateBadge / DcrStateBadge / ImportStatusBadge / CoverageBadge / ConfidenceCell / the lifecycle steppers all pair a token colour + a non-colour glyph + a text label + an aria-label, so state survives colour-blindness, greyscale audit-export, and screen readers. This is the load-bearing DP-5/WCAG bar the product is CI-gated on, and on the components that matter most it is genuinely met.
- Restraint is held with discipline app-wide: one indigo accent, layered neutral surfaces, an 8px scale, a system-font stack (air-gap-safe by design), and a total absence of every named anti-reference — no gradients, eyebrows, side-stripes, hero-metric templates, or numbered scaffolding. The app reads as 'the standard made operable', which is the stated north star, and would not make a Linear/Stripe-fluent user flinch.
- Compliance-grade safety mechanics are consistent and honest: capability flags computed server-side so write affordances never show-then-403; close/release gates that defer to the server's authoritative 409 and surface exactly what's missing in plain language; per-mount idempotency keys against double-submit; SoD self-action caught with bespoke human copy; the ingestion included_candidate guard that makes a bad bulk-sweep structurally impossible; and forbidden reads that degrade to a calm named-permission panel rather than a crash.
- Inline microcopy and permission honesty are best-in-class for the domain: forbidden panels name the exact permission key AND the roles that hold it (turning a dead end into self-service), and every lifecycle action carries a one-line plain-language explainer of its effect ('Freezes the minutes and starts approval', 'flips the 9.3 ★') — the calm/reassuring brand voice delivered without jargon theatre.

## Priority issues
- [P1] Two non-interoperable colour conventions render different hues for the same semantic state, and ~12 near-identical badge components each redefine their own glyph map (amber = ▲ here / ◔ there; the Objectives register RAG badge has no glyph at all).
  - Why: In a compliance product where red/amber/green is load-bearing, two greens or two reds that don't match in the SAME table row makes the external auditor (Olsen) ask which status is authoritative — the precise ambiguity DP-5 forbids. The copy-paste-per-slice badge proliferation is the root cause that keeps the glyph and colour vocabularies drifting, so consistency degrades with every new feature. This is the single highest-reach defect in the app.
  - Fix: Register the --es functional hues as Mantine virtual colours (so 'green'/'amber'/'red' resolve to the AA-tuned tokens) OR convert every palette-name consumer to var(--es-*) — pick ONE. Then extract a single <StatusBadge token glyph label/> primitive over ONE canonical glyph-per-tone map, and route every feature's META through it. Add the missing glyph to the Objectives register RAG badge.
  - Command: /impeccable distill | Surfaces: Design system, Objectives, Home, CAPA, Audits, Management Review, Compliance/Drift, Documents
- [P1] Raw backend enum tokens and machine field keys are rendered verbatim to the Quality Manager and external auditor.
  - Why: CAPA shows 'RootCause'/'ActionPlan'; DCR shows 'effectivity_transition'/'records_produced_under' + raw uppercase MAJOR/MINOR; MR's SummaryTable does k.replace(/_/g,' ') → 'verify failed at <timestamp>'; Drift dumps snake_case counts in mono; the inbox shows 'action:<uuid>'. A schema leaking into the UI is the legacy-QMS anti-reference the product exists to replace, and it directly undercuts 'legibility is the feature' on the most-scrutinised surfaces. The pattern of exhaustive label maps existing elsewhere proves these are omissions, not decisions.
  - Fix: Add total label-Record maps for every user-facing enum (CLOSE_STATE_LABEL, DIMENSION_LABEL, SIGNIFICANCE_LABEL, humanised drift count keys, humanised stage_key), mirroring the SEVERITY_LABEL/INPUT_LABEL pattern already in the codebase; for the MR SummaryTable, curate per-input-type human rows instead of iterating Object.entries.
  - Command: /impeccable clarify | Surfaces: CAPA, DCR, Management Review, Compliance/Drift, Documents/Tasks
- [P1] Irreversible compliance actions fire on a single click with no confirmation and no undo, app-wide.
  - Why: NCR disposition (one-shot ISO 8.7), document Release, MR Submit/Release/Close + Remove-output (deletes a recorded 9.3.3 decision), audit Issue-report/Close, CAPA Return-to-root-cause, and ingestion bulk-Exclude all mutate audit-grade records with a bare button. A misclick by Ken or Mara is unrecoverable through the UI and writes a permanent record — the highest error-prevention bar this product must meet, currently unmet everywhere except where a modal already gates the flow.
  - Fix: Add ONE shared ConfirmDestructive primitive that restates the consequence (reusing the helper-text copy already written), and gate every irreversible transition behind it; offer an inline Undo (post the inverse decision via the existing idempotency-key path) where the data model allows.
  - Command: /impeccable harden | Surfaces: CAPA, Documents, Management Review, Audits, Ingestion
- [P1] Decorative emoji used as functional icons.
  - Why: TopBar action affordances (🔍 search, ⌖ Tasks, 🔔 Acks, 👤 Account via &#…; entities) and ingestion kind/plan glyphs (📄 Document, 🔒 Record, 🛡 plan) are an explicit PRODUCT.md anti-reference, render as OS-specific pictographs that break the air-gap 'renders identically on a disconnected box' promise, and clash with the disciplined geometric-glyph set used by every other badge. The auditor scrutinising for audit-grade seriousness sees 🔔.
  - Fix: Replace the emoji with a single drawn icon set (Tabler/@tabler/icons-react, common with Mantine, or inline SVGs consuming es-* tokens) at a consistent stroke weight; for the ingestion kind glyphs reuse the geometric vocabulary. Keep existing aria-labels.
  - Command: /impeccable shape | Surfaces: App shell, Ingestion
- [P1] The surfaces the power users live in most can't be triaged in place — TasksInbox/Home rail and the DCR/MR registers name no subject, and almost no register has search/sort/bulk/keyboard affordances.
  - Why: Mara and Ken process work in batches daily; the general task queue shows action+stage+due with no document identity, so every task is a click-through to learn what it is, and the DCR/MR/Audits/Objectives registers are unsortable, unsearchable, bulk-less flat lists. This is the single biggest efficiency tax on the core approve/triage journeys and the largest gap against the daily-power-user persona. The Library density toggle and ingestion bulk bar prove the capability exists; it just isn't applied to the high-traffic surfaces.
  - Fix: Enrich the /tasks list (and DCR/MR list serializers) with a subject identifier + short title; add a reusable subject column, debounced text filter, sortable columns, and keyboard row-nav (up/down + Enter), reusing the Library/CAPA filter patterns; extend the DOC_ACK bulk-inbox pattern to homogeneous task types.
  - Command: /impeccable optimize | Surfaces: Tasks/Review, Home, DCR, Management Review, Audits, Objectives
- [P2] No freshness / 'as of' timestamp on any status board, and timestamps that do appear are timezone-ambiguous (iso.slice(0,16)).
  - Why: Home, Objectives, Compliance and Drift present live-looking status with no indication of when it was computed; the drift scan times are unlabelled UTC-or-local. Olsen needs provable currency ('the scan ran today') and Mara needs to trust the numbers before acting — in the one product whose entire thesis is proving currency, a status board without a clock invites a stale-data trust failure.
  - Fix: Surface a single localised, timezone-explicit 'as of <relative time>' in each board header, driven by the newest underlying read or the server-reported scan/computed-at (DriftStatus already carries finished_at); flag a scan as stale when older than the expected interval.
  - Command: /impeccable clarify | Surfaces: Home, Objectives, Compliance/Drift
- [P2] No shared loading / error / empty primitives, and almost no error Alert offers a Retry button.
  - Why: Loading is a bare centred Loader on most pages but content-shaped skeletons only in Home tiles (two status languages for one concept), and the duplicated 'Couldn't load — Please try again' copy gives no button to actually retry — the user must reload the whole SPA on any transient blip. On a self-hosted box one slow endpoint shouldn't strand a surface, and the inconsistency reads as a multi-author system at exactly the moments (failure) when trust matters most.
  - Fix: Add shared <PageLoading> (skeleton-of-layout), <ErrorState onRetry> (Retry wired to the query refetch), and <EmptyState> primitives; replace the per-page Loader/Alert blocks so loading→error→empty is one vocabulary app-wide. Also fix useAckCount to distinguish error from zero (the silent false all-clear).
  - Command: /impeccable harden | Surfaces: App shell, Home, Objectives, CAPA, Compliance/Drift, Audits, DCR, Documents
- [P2] Badge contrast relies on Mantine's light-variant math over a raw CSS-var hue rather than the AA-tuned -soft/-text token pairs the design system defines.
  - Why: Every status badge passes color="var(--es-*)" with variant="light", so Mantine computes the background/text tint rather than using the hand-tuned --es-success-text-on-success-soft pairing — an unverified WCAG 2.2 AA contrast risk on the most repeated component in the app (every list row, header, stepper-adjacent badge), invisible to tests. Amber/warning text on a light amber fill is the classic failure case.
  - Fix: Either register the tokens as Mantine theme colours with a proper shade ramp, or set the Badge background/color explicitly from the --es-*-soft / --es-*-text token pairs via the styles API; audit each variant in light and dark for ≥4.5:1.
  - Command: /impeccable audit | Surfaces: Design system, Documents, CAPA, Objectives, Management Review, Compliance/Drift, DCR, Ingestion
- [P3] The ISO PDCA structure is applied only to the clause section; the ~11 top-level rail items are an ungrouped flat CRUD list, and there is no in-product help/legend layer.
  - Why: PRODUCT principle #1 calls for IA that flows the way ISO 9001 flows, but the chrome reads as 'a database with modules' above the clause section, forcing a long linear scan (>4-per-group). And the glyph vocabulary + domain terms (Drift, effectivity_transition, disposition tokens) have no legend/tooltip, so Sam and Olsen meet them cold. Both are second-order to the P1s but both touch every page.
  - Fix: Fold the static destinations under PLAN/DO/CHECK/ACT headers (the code already renders these for clauses), and once a canonical glyph map exists, surface it as a small legend Popover on the registers/checklist plus tooltips on domain terms.
  - Command: /impeccable layout | Surfaces: App shell, all registers

## Cross-surface themes
- RAG/status vocabulary is NOT consistent across surfaces — and this is the app's defining flaw. Two colour systems coexist (CSS-var tokens vs Mantine palette names) producing mismatched hues for the same state in a single row; the glyph language drifts (amber = ▲ vs ◔; some RAG badges carry a glyph, the Objectives register's carries none); and ~12 hand-rolled badge components prevent convergence. The core StateBadge family is exemplary — the rest of the app has not been refactored toward it.
- Raw backend enums/keys leaking to users is a recurring, multi-surface 'shipped the data model' smell (CAPA close_state, DCR impact dimensions + significance, MR SummaryTable, drift counts, inbox stage_key) — striking because the SAME codebase has exhaustive humanising label maps elsewhere, proving these are unfinished omissions rather than a house style.
- Irreversibility is handled inconsistently: where a modal already gates a flow the act feels deliberate, but every bare-button irreversible transition (disposition, release, MR submit/close, audit close, bulk-exclude) ships with no confirm and no undo — there is no shared destructive-action pattern, so safety depends on which slice happened to wrap its action in a modal.
- Error/loading/empty states are individually calm and on-brand but NOT standardised — hand-rolled per page, with a split loading language (Loader vs Skeleton) and near-total absence of a Retry affordance. The recovery COPY is uniformly excellent; the recovery MACHINERY (shared primitives, retry buttons) is uniformly missing.
- Power-user efficiency is present in isolated pockets (Library density toggle, ingestion bulk bar + ⌘K keyboard nav, redline n/p) but absent from the high-traffic registers and the general task queue — the team clearly CAN build these affordances, they just haven't been applied where Mara and Ken actually live, so the app rewards the workflows it was prototyped on and taxes the ones used daily.
- Motion is uniformly restrained-to-absent and correctly reduced-motion-guarded — consistent, on-brand, no surface over- or under-animates. The calm aesthetic and token discipline are the one thing that IS uniform app-wide.
- Permission/forbidden handling and capability-gating are consistently excellent across every surface — the deny-by-default posture is expressed legibly in the IA and the affordances, with no show-then-403 and named-permission degradation everywhere. This is the most uniformly strong cross-surface theme.

## Minor observations
- Breadcrumb LABELS map covers only a handful of segments; unmapped routes (/management-reviews, /dcrs, /objectives) fall back to the raw URL slug ('management-reviews') as the crumb label.
- RAG badge labels on Home are the literal colour word ('Green'/'Amber'/'Red'), not the meaning ('On track'/'Needs attention'/'Action required') — the colour-safe channel exists but carries no meaning for a greyscale reader.
- Several detail-page metric tiles emit '—' filler captions ('history not in scope','Not yet effective') for not-yet-effective docs, reading as missing data rather than expected Draft state.
- DocumentDrawer tabs (Overview/History/Where-used) silently differ from the full-page tabs (adds Approvals/Acks), so 'Open full page' changes the available IA without a note.
- ContentBlock (CAPA) and similar fall back to raw JSON.stringify for nested objects, risking an unstyled blob dumped into the auditor's evidence trail.
- Programme-page row selection (data-selected on the <tr>) has zero matching CSS — clicking a programme gives no highlighted-row feedback, only a distant heading change.
- Audit findings render as a Paper-card Stack while the same surface uses sortable Tables elsewhere — a 20+ finding audit becomes an unsortable scroll.
- The skip-link uses inline left:-9999px rather than the token/utility system, and DetailDrawer width resets per mount with no persistence.
- ConfidenceCell maps both LOW and AMBIGUOUS to --es-danger, leaning entirely on the text label to disambiguate two distinct outcomes.
- Acks Indicator and DcrStageTimeline reimplement deep-links/date-formatting that a shared constant or util would keep from drifting.
- The dead disabled Tasks Indicator in the TopBar is a non-functional stub sitting next to the working Acks bell — users will try it and erode trust in the other status lights.
