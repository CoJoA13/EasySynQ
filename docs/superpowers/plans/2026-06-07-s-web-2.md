# S-web-2 Implementation Plan — Faceted Library + read-only detail drawer

> Spec: `docs/superpowers/specs/2026-06-07-web-track-s-web-2-design.md`. Branch: `feat/s-web-2-library`.
> Method (S-web-1 precedent): spec+plan → TDD per task (test first where logic exists) → per-task quality review →
> whole-branch diff-critic before PR. CI: all five jobs green. **No migration** (head stays `0044`).

## Build sequence (dependency order)

### Phase 0 — backend read endpoints (no migration)

**T0.1 — `GET /document-types`.** New `apps/api/src/easysynq_api/api/document_types.py`: **authenticated-only**
(`get_current_user` — matching `GET /documents`; `require("document.read")` resolves at SYSTEM scope and would
exclude ordinary readers), org-scoped list of `{id, code, name, document_level, is_singleton}` ordered by `name`.
Register the router in the app factory (alongside `clauses`). *Test:* `tests/integration/test_document_types.py`
(DB-backed → integration, not unit) — 401 unauth, org-scoped, shape, seeded SOP. *Contract:* add to `openapi.yaml`.

**T0.2 — `GET /directory/users`.** New `apps/api/src/easysynq_api/api/directory.py`: **authenticated-only**, returns
`[{id, display_name}]` for ACTIVE non-guest users, org-scoped, ordered by `display_name`. *Test:*
`tests/integration/test_directory.py` — 401 unauth, **asserts e-mail/keycloak_subject/status/roles are NOT present**
(minimal-disclosure), guest/disabled excluded, self present. *Contract:* `openapi.yaml`.

**T0.3 — `GET /documents` pagination envelope + date filter** (`api/documents.py`).
- Add `offset: int = 0`; keep `limit` (default 50, max 100). Introduce `_LIST_SCAN_CAP = 2000`.
- Refactor `list_documents`: fetch candidates (filters + kind=DOCUMENT + org) `ORDER BY created_at DESC LIMIT
  _LIST_SCAN_CAP`; authz-filter in Python (existing loop); compute `authorized`; `page_rows =
  authorized[offset:offset+limit]`; `has_more = len(authorized) > offset + limit`. Return
  `{"data": [_document(d, clause_refs=…) for d in page_rows], "page": {"limit", "offset", "returned":
  len(page_rows), "has_more"}}`. Batch the clause_refs over `page_rows` only.
- Add `("effective_from","gte")` + `("effective_from","lte")` to `_FILTER_ALLOW`; in `_filter_condition`, build an
  `EXISTS`/join condition over `DocumentVersion` where `DocumentVersion.id ==
  DocumentedInformation.current_effective_version_id AND effective_from >=/<= <parsed ISO>` (422 on a bad timestamp).
- *Tests:* unit (`test_documents.py`) — envelope shape, `offset`/`limit` slicing, `has_more`, date-filter parse +
  422; **integration** (`tests/integration/test_vault.py` or a new file) — a **scoped** (non-admin) user paginates
  correctly under the per-row authz filter, and the `effective_from` bound selects the right docs. Run-scoped/delta
  assertions (shared integration DB). *Contract:* update the `GET /documents` response to the `{data,page}` envelope
  + the new params in `openapi.yaml`.

Phase-0 gate: `/check-api` + `/check-migrations` (no-op confirm) + `redocly lint`.

### Phase 1 — front-end foundation

**T1.1 — types** (`apps/web/src/lib/types.ts`): add `DocumentType`, `UserDirectoryEntry`, `DocumentsPage`
(`{data: DocumentSummary[]; page: {limit; offset; returned; has_more}}`), `DocumentVersion`, `WhereUsed` (+ its
bucket types), `DocumentFilters` (the typed facet state). Add the friendly fields the drawer needs. `import type`.

**T1.2 — hooks** (one file each, S-web-1 idiom): `features/library/useDocuments.ts` → take `(filters, {limit,
offset})`, build the bracketed query string, return `DocumentsPage`; `app/shell/useDocumentTypes.ts` +
`app/shell/useUserDirectory.ts` (shell-scoped, cached id→name maps); `features/library/useDocumentVersions.ts` +
`features/library/useWhereUsed.ts` (drawer tabs, `enabled` on tab activation). Query keys include params.

**T1.3 — MSW fixtures/handlers** (`src/test/msw/handlers.ts`): fixtures + default handlers for the new/changed
endpoints (envelope shape, document-types, directory, a single document, versions, where-used) — full-UUID ids,
ISO timestamps, in lockstep with `lib/types.ts`.

### Phase 2 — the Library page (`features/library/`)

**T2.1 — `ClauseTree.tsx`** — PDCA-banded clause-spine tree from `useClauses` (reuse the LeftRail grouping logic),
`★` marker, selecting a clause sets the Clause filter; ARIA tree/`aria-current`. (No counts.)

**T2.2 — `FacetBar.tsx` + `FilterShelf.tsx`** — five facet controls (Type/Status/Owner/Clause/Effective-date) +
removable active-filter chips + Clear-all; **URL query-string is the source of truth** (`useSearchParams`), mapped
to/from the bracketed API filters. Status/Clause enumerations are static/`useClauses`; Type/Owner from the cached
maps; Effective-date = relative buckets → `effective_from[gte]`.

**T2.3 — `LibraryPage.tsx` (rebuild)** — two-column layout (ClauseTree | right column); results table with the
§5.3 columns (resolve Type via the types map, Owner via the directory map, State via `StateBadge`); density toggle;
loading skeletons / empty-with-filters / permission-empty; keyboard rows; result-count caption from `page`. Row →
`?detail=<id>`.

**T2.4 — `Pagination.tsx`** — page-size (25/50/100) + prev/next from `offset` + `page.has_more`; writes `offset`
to the URL.

**T2.5 — `StateBadge`** — confirm the full 7-state family (Draft/InReview/Approved/Effective/UnderRevision/
Superseded/Obsolete) each with icon + label (color-never-alone); extend if S-web-1 shipped a subset.

### Phase 3 — the shared header + tabbed drawer

**T3.1 — `features/document/ArtifactHeader.tsx`** (new shared, lens-agnostic, DP-5): identifier · title · large
state badge · resolved owner/type · clause chips · effective date. Reused by S-web-3. **Avoid the S-web-1
StateBadge↔Drawer import cycle** — header imports `StateBadge` only; the drawer imports the header (one direction).

**T3.2 — `DocumentDrawer.tsx` (rebuild to tabbed)** — Mantine `Tabs`, `?detail=<id>` (+ `&tab=`), lazy per-tab
queries, header always renders. **Overview** (header + control-metadata `<dl>` + controlled-copy download when an
effective version exists), **History** (`useDocumentVersions` timeline; 403→quiet no-access), **Where-used**
(`useWhereUsed` §7.2 categories). Cold-deep-link guard (resolve from loaded list or fetch `GET /documents/{id}`).

### Phase 4 — wiring, tests, a11y, gate

**T4.1** — confirm routing/Breadcrumb/LeftRail already cover `/library` (S-web-1) — no new route needed; ensure the
drawer deep-link round-trips.
**T4.2** — component/integration tests per Phase 2/3 unit (facets+URL, table resolution, pagination, drawer
deep-link + tab lazy-load + 403 path) — each page/drawer test includes **jest-axe**.
**T4.3** — full local gate: `cd apps/web && npm run lint && npm run typecheck && npm run build && npm test`;
`/check-api`; `redocly lint`; `/check-migrations` (no-op).

### Phase 5 — review, live smoke, PR

**T5.1** — `diff-critic` subagent on the whole branch diff (WORM/authz/run-scoped-assertions lenses; here especially
the **authz-correct pagination** + the **directory minimal-disclosure** + the **read_draft gate** on History).
**T5.2** — live smoke against the running stack (`http://localhost`, `demo`): the Library renders real docs with
resolved Type/Owner, facets filter + chips, pager works, drawer tabs lazy-load.
**T5.3** — `/pr` → PR against protected `main`; all five CI jobs green → squash-merge.

## Risks / watch-items

- **Pagination correctness under authz** is the load-bearing backend change — the integration test MUST exercise a
  **scoped** user (not just the SYSTEM-override admin, who sees all and would mask a bug).
- **Response-shape change** on `GET /documents` (bare array → `{data,page}`) — update every consumer (web hook,
  MSW fixtures, openapi.yaml) in the same PR; grep for other callers first.
- **Directory disclosure** — assert in tests that only `{id, display_name}` is returned; gate `document.read`.
- **Import cycles** (S-web-1 lesson) — header→StateBadge→(nothing); drawer→header; page→drawer. Shell never imports
  features.
- **MSW `onUnhandledRequest:"error"`** — every endpoint a screen hits needs a default handler.
- **`document.read_draft`** — the History tab can 403 for a low-priv reader; render quiet absence, don't crash.
