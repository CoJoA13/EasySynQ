# S-web-4b Implementation Plan — the worker-async visual page-image diff viewer

> Spec: `docs/superpowers/specs/2026-06-08-web-track-s-web-4b-design.md`. Branch: `feat/s-web-4b-visual-diff`.
> Method (S-web-1…4 precedent): spec+plan → TDD per task (test first where logic exists) → per-task quality
> review → whole-branch `diff-critic` before PR. CI: all five jobs green. **Front-end only — no migration (head
> stays `0044`), no new permission key, no `openapi.yaml` change** (the worker-async backend trio is built +
> contracted, S-dcr-3a/3b). Locked decisions: **D-A** single-pane + from/to/diff toggle (authed
> fetch→blob→objectURL→`<img>`); **D-B** the Text|Visual switch in `VersionCompare`, `RedlineViewer` byte-identical;
> **D-C** full viewer (rail + toggle + `n`/`p` + phased poll + jest-axe gate).

## Build sequence (dependency order)

throws `ApiError(403)`; a 404 throws `ApiError(404)` with the problem `code`.

**T0.3 — jsdom objectURL stub** (`apps/web/src/test/setup.ts`): add `URL.createObjectURL = () => "blob:mock"` +

**T0.4 — MSW fixtures + handlers** (`apps/web/src/test/msw/handlers.ts` — extend, keep existing tests green): add

`http.post("/api/v1/documents/:id/versions/:vid/visual-diff", () => HttpResponse.json(visualDiffFixture))` (default

### Phase 1 — the hook (the async-poll core)

**T1.1 — `useVisualDiff(documentId, toVid, fromVid, enabled)`** (`features/document/useVisualDiff.ts`, spec §5.2)
— the **POST-trigger + GET-poll** split:
- a `useMutation` POSTing `…/versions/${toVid}/visual-diff?from=${fromVid}` (idempotent); fired once per enabled
  distinct pair via a `useEffect` (deps: ids + `enabled`) and on `retry()`; `onSuccess` seeds the poll cache
  (`queryClient.setQueryData(key, data)`);
- a `useQuery` GET-poll, key `["visual-diff", documentId, toVid, fromVid]`, `enabled` once the POST has run,
  `refetchInterval: (q) => q.state.data?.status === "Pending" ? 2500 : false` (stop at terminal — the
  `SetupWizard.tsx:64-108` precedent);
- returns `{ status: poll.data ?? mutation.data, isPending, isError, error, retry }`. `enabled` only when
  `documentId && toVid && fromVid && toVid !== fromVid` **and** visual mode is active.

*Test:* `useVisualDiff.test.tsx` (renderHook + the MantineProvider/QueryClient/AuthContext wrapper, the
`useVersionDiff.test.tsx` precedent) — disabled ⇒ no POST; POST(Pending)→poll→`Ready` (stops polling); `Failed`;
`Unavailable`; `403 ⇒ ApiError(403)`; `retry()` re-POSTs.

### Phase 2 — the viewer (the meaty new surface)

**T2.1 — `VisualDiffViewer.tsx`** (`features/document/`, spec §6.1) — props `{documentId, fromVid, toVid}`; uses
`useVisualDiff`. Local view state: `page` (0-based index, default the first **changed** page else 0) + `layer`
(default `"diff"`). Render the state machine:
- **403** → quiet "You don't have access to the visual diff." (DP-6, the `RedlineViewer.tsx:40-46` precedent).
- **Pending** → §4.9 skeletons matching the final layout (rail + pane) + a phase label + an `aria-live="polite"`
  region; the rest of the page interactive; a manual **"Re-request render"** (→ `retry()`) for the dev
  renderer-off "Pending forever" case.
- **Failed** → a calm `Alert color="red"` with `reason` + the source-download fallback, **no Retry** (a terminal `Failed` row is not re-drivable — the backend re-enqueues only a `Pending` row; the re-request affordance is on `Pending` only). [Codex P2 fix.]
- **Unavailable** → calm `Alert` "Visual diff unavailable — {reason}" + per-version **source-download** links
  (the `openSource` idiom from `RedlineViewer.tsx:74-83`).
- **Ready** → the **rail** (labelled list of `pages[]`; 1-based "Page N" label over the 0-based index; a
  **non-color** glyph+label "changed" when `pages[i].changed`; keyboard-navigable; the SR change index) + the
  **pane** (an `<img>` via a small `usePageImage(documentId, toVid, fromVid, page, layer)` helper or an inline
  effect that `getBlob`s the page URL → `URL.createObjectURL` → `src`, **revoking** the prior objectURL on
  page/layer change + unmount; meaningful `alt` "Page N of M — {layer} layer ({changed|unchanged})"; a 404 from
  the helper ⇒ a "No image on this side for this page" note) + the **layer toggle** (`SegmentedControl
  from|to|diff`, default `diff`) + **`n`/`p`** next/previous-**changed**-page nav + an `aria-live` region for the
  current page/layer + the **footer-band caveat** note.

*Test:* `VisualDiffViewer.test.tsx` — Ready: the rail with non-color changed markers; the pane `<img>` (assert the
authed `getBlob` carried `Authorization`, the objectURL `src`, the `alt`); the layer toggle re-fetches with
`?layer=to`/`from`; the 404 no-image note; `n`/`p` jumps changed pages; Pending skeleton + `aria-live` phase (no
frozen UI); Failed banner + Retry (re-POST); Unavailable source-download (`window.open`); 403 quiet; **jest-axe**
on Ready + Pending + Failed + Unavailable.

## Risks / watch-items

- **The worker-async contract is the load-bearing invariant.** The **GET poll must never enqueue** and must
  **never fire before the POST** (404-before-POST) — hence the POST-trigger seeds the poll cache. `refetchInterval`
  must **stop at any terminal status** (Ready/Failed/Unavailable), never spin forever. POST is idempotent
  (`UNIQUE(from,to)` + enqueue-only-while-Pending), so a duplicate POST is safe and re-drives a stalled Pending.
- **Authed `<img>` + objectURL leak.** The page endpoint is **authenticated, not presigned** — a bare `<img src>`
  401/403s; fetch with the bearer → blob → objectURL, and **`URL.revokeObjectURL` on every page/layer change and
  on unmount** (the only authed-binary proxy in the SPA — every other download is a presigned URL, D1).
- **`Unavailable` and a no-image layer are NORMAL, not errors.** `Unavailable` = a non-renderable version (R26) →
  calm fallback, no retry. A page-endpoint **404 "No image for this page/layer"** is expected for an added page's
  `from` layer / a removed page's `to` layer → a per-layer note, not a broken image or a banner.
- **Dev renderer-off ⇒ `Pending` forever.** A Gotenberg outage leaves the row `Pending` (not Failed) — the viewer
  must poll patiently + offer a manual re-request, and the smoke must exercise this (the renderer is often off in
  dev). Don't assume Pending self-resolves.
- **Footer/watermark band false-positive.** The watermarked rendition's footer differs by revision → a
  near-universal "changed" footer region. The copy must not over-claim page-level precision (the v1 caveat; raw
  render is v1.x).
- **`RedlineViewer` parity.** The Text path + `RedlineViewer.test.tsx` must stay **byte-identical** — the switch
  lives in `VersionCompare`, never inside `RedlineViewer` (the engineering-patterns "keep the tested path
  unchanged" discipline).
- **0-based page index.** `pages[].page` and the page route are **0-based**; the rail must label 1-based ("Page 1"
  = `page 0`). Requesting `page = page_count` 404s.
- **MSW `onUnhandledRequest:"error"`** — the new visual-diff POST/GET/page handlers must exist as defaults, in
  lockstep with `lib/types.ts`, or page/viewer tests fail. The page handler returns binary (`HttpResponse(bytes,
  image/png)`); the objectURL stub (T0.3) is required.
- **a11y release gate.** jest-axe + eslint-jsx-a11y on the new viewer: non-color changed markers (glyph+label), a
  meaningful `alt` per page image, the rail as the SR change index, the layer toggle as a real radio/tablist,
  `n`/`p` keyboard, a live region for the poll, ≥24px targets, visible focus, `prefers-reduced-motion`.
- **Scope guard.** Single-pane toggle this slice — **no** two-pane synced-scroll, **no** canvas/onion-skin, **no**
  client zoom/pan, **no** Beat reaper. Those are deferrals (spec §10), not this PR.
- **Zero backend drift.** If the smoke reveals a genuinely missing read (none expected — the trio is contracted),
  it is its own change, not folded silently into this front-end slice.
