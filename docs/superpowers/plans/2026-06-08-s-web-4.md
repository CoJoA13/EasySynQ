## Build sequence (dependency order)

### Phase 0 — types, fixtures, the diff hook (the data foundation)

**T0.1 — types** (`apps/web/src/lib/types.ts`, `import type`): add `DiffProvenance` (`{version_id, version_seq,
revision_label, version_state, change_significance, change_reason, effective_from, effective_to, author_user_id,
created_at, signatures: {meaning, signer_user_id, signed_at}[]}`); `MetadataDiffEntry` (`{field, from: unknown,
to: unknown, changed: boolean}`); `TextDiffHunk` (`{op: "equal"|"insert"|"delete", text: string}`); `TextDiff`
(`{status: "ok"|"unavailable", reason?: string, hunks?: TextDiffHunk[]}`); `VersionDiff` (`{document_id, from:
DiffProvenance, to: DiffProvenance, metadata_diff: MetadataDiffEntry[], text_diff: TextDiff}`); `DocumentDownload`
(`{download_url: string, content_type?: string, rendition: "controlled_copy"|"source"}`). Mirror `openapi.yaml`
exactly (`:4801-4878, 5005-5015`).

**T0.2 — MSW fixtures + handlers** (`test/msw/handlers.ts` — extend, keep existing tests green): add
`capabilities` to the detail path of `GET /documents/:id` (a detail fixture or augment `docFixture[0]` with
`{checkout:false,edit:false,manage_metadata:false,submit:false,release:false,obsolete:false,read_draft:true}` so
the default page renders read-only + history); a `diffFixture` (`VersionDiff` with a `text_diff.status:"ok"`
hunks array — equal+insert+delete — and a changed `metadata_diff` row); add handlers `GET
/api/v1/documents/:id/versions/:vid/diff` (200 `diffFixture`; the test overrides for the `"unavailable"` +
403 cases) and `GET /api/v1/documents/:id/download` (`{download_url:"https://minio.test/cc", content_type:
"application/pdf", rendition:"controlled_copy"}`). NB the existing default `me/permissions` handler returns
**no** permissions (`handlers.ts:295-297`) → the page renders read-only by default, which is what most page
tests want.

**T0.3 — `useVersionDiff`** (`features/document/useVersionDiff.ts`): `useQuery` over `GET
/documents/{id}/versions/{toVid}/diff?from={fromVid}`, key `["version-diff", documentId, toVid, fromVid]`,
`enabled` only when `documentId && toVid && fromVid && toVid !== fromVid`. Returns `VersionDiff`. *Test:*
`useVersionDiff.test.tsx` — fetches with both ids; disabled when a pair is missing/equal; a 403 surfaces as
`ApiError(403)`.

### Phase 1 — the reusable extraction + the rendition card (low-risk, test-backed)

**T1.1 — extract `ControlMetadata` + the download helper from `OverviewTab`.** Pull the metadata `<Table>` into

### Phase 2 — the redline viewer (the meaty new surface)

**T2.1 — `RedlineViewer.tsx`** (`features/document/`): props `{documentId, fromVid, toVid}`; uses
`useVersionDiff`. Renders the §6.2 header (`from.revision_label → to.revision_label` + the newer's
`change_reason` + author/state from `DiffProvenance`); the **metadata-diff table** (only `changed` rows, `null`
→ "—"); the **text redline** — `status:"ok"` → ordered hunks (`equal` muted context; `insert` `--es-success` +
underline + leading `+`; `delete` `--es-error` + strike + leading `−`; **non-color-only** via the markers +
`ins`/`del` semantics); `status:"unavailable"` → "Text redline unavailable — {reason}" + a source-download
fallback. Keyboard `n`/`p` → next/prev changed hunk; a labelled change **index** for SR users; a labelled region;
focus-on-open; `prefers-reduced-motion`. **403 → quiet** "no access to the redline." *Tests:* `RedlineViewer.test.tsx`
— the changed-only metadata table; the ins/del/equal rendering with the `+`/`−` markers (assert markers, not just
color); the `"unavailable"` fallback; the 403-quiet path; `n`/`p` nav; the change index; **axe**.

**T2.2 — the Compare control** (on the History card; `VersionCompare.tsx` or inline in the page): two Mantine
`Select`s from `useDocumentVersions`, default `from`=prior / `to`=governing-or-newest; on change write

when both are present. The drawer's `HistoryTab` stays **untouched** (diff is page-only). *Tests:* picking a pair

## Risks / watch-items

- **No-dead-button discipline (DP-6)** is the load-bearing UX invariant: `AuthorActions` already handles
  capability/state/lock gating (reused verbatim); the **Export** button must gate on `can("document.export")`
  (quiet-absent by default); **history + diff** must render 403 as quiet "no access," never an error banner (the
  `HistoryTab.tsx:11-18` precedent). The page itself 404s/403s as scoped empty states with the header still up.
- **Redline status-never-color-only** — `insert`/`delete` MUST carry the `+`/`−` markers + `ins`/`del`
  semantics, not color alone (WCAG 2.2 / DP-7); the test asserts the markers. Honour the backend's **line-level**
  hunks (not word-level — the contract says line-LCS); word-level is a later enhancement.
- **`text_diff: "unavailable"`** (Tika down / non-extractable / binary) is a **normal** path, not an error —
  render the calm fallback + source-download links (the visual diff that fully covers binary docs is S-web-4b).
- **`OverviewTab` parity** — the `ControlMetadata`/download extraction must leave the drawer's `OverviewTab`
  rendering byte-identical (its test is the backstop); don't change the drawer's behaviour while refactoring.
- **No new import cycle** — `RedlineViewer`/`RenditionCard`/page live in `features/document/` and may import
  `features/authoring/AuthorActions` (mirroring `DocumentDrawer.tsx:7`); `AuthorActions` imports nothing from
  `features/document/` (verified) — keep it that way.
- **Routed-page testing** — render the page inside `<Routes>` with the `route` param (`test/render.tsx`'s
  `MemoryRouter`), since it reads `useParams`; a bare render without a route has no `:id`.
- **MSW `onUnhandledRequest:"error"`** — the new `…/diff` and `…/download` handlers must exist as defaults, or
  page tests fail; keep them in lockstep with `lib/types.ts`.
- **Scope creep guard** — the visual page-image diff is **S-web-4b**; Approvals/Acks/Audit/next-review are
  honest deferrals (§10). Do not start them here.
- **Zero backend drift** — if the smoke reveals a genuinely missing read (none expected), it follows the
  own-data pattern (migration-free read, no new key) and is its own change — not folded silently into this slice.
