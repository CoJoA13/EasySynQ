## PR 1 — S-risk-4a (`feat/s-risk-4a`) — `GET /risks/summary` (BE, small)

**Files:**
1. `apps/api/src/easysynq_api/api/risk.py` — add the route **in the `/risks/register*` static-route
   block, BEFORE `@router.get("/risks/{risk_id}")`** (S-pack-2 ordering). Gate on a module-level
   `_register_read_system = require("register.read")` (default SYSTEM scope). Handler:
   ```python
   @router.get("/risks/summary")
   async def risk_summary_endpoint(
       caller: AppUser = Depends(_register_read_system),
       session: AsyncSession = Depends(get_session),
   ) -> dict[str, Any]:
       governing = await governing_register(session, caller.org_id)
       if governing is None:
           empty = summarize_register({"rows": [], "criteria": default_criteria(ScoringMethod.MATRIX_5X5)})
           return {"published": False, **empty}
       return {"published": True, **summarize_register(governing)}
   ```
   Import `summarize_register` (`..domain.risk.summary`), `default_criteria` (`..domain.risk.rules`),
   `ScoringMethod` (already imported). **Body-first to dodge the ruff just-added-import strip** (recurring).
   *(The empty-criteria path: `summarize_register` calls `resolve_criteria(register, method)` per row;
   with zero rows it never touches criteria, but pass a valid `default_criteria` snapshot defensively so
   the helper's contract — a `{rows, criteria}` dict — holds.)*
2. `packages/contracts/openapi.yaml` — a `GET /risks/summary` path under `tags: [risk]`, operationId
   `getRiskSummary`, gated `register.read`; a new `RiskRegisterSummary` schema (`additionalProperties:
   false`, required `[published, total, by_band, high_risk, by_type, effectiveness]`); `by_band` keys
   `critical/high/medium/low/unscored`; `by_type` keys `risk/opportunity`; `effectiveness`
   `treated/recorded/pending`. Mount near the existing `/risks/register` block.
3. `apps/api/tests/integration/test_risk_summary.py` (new) — the 4 tests (§2 of the spec): pre-release
   `published:false`+zeros; post-release `published:true`+governing counts; no-grant → 403; the
   read-of-record UnderRevision-invisible proof. Reuse `test_risk_lifecycle.py` scaffolding
   (`_drive_to_editable`, `_create_risk`, publish, `_approve_and_release`, the `restore_register_head`
   teardown + normalize-at-start; `_auth`/`_grant`). Assert **deltas / this-run rows**, never absolutes
   (the shared-DB discipline).

**Gate:** `/check-api` (ruff + mypy-strict + targeted unit) + `/check-contracts` (redocly). Integration
runs in CI (Windows-local `-m integration` can't run; `-m unit` has the 17-failure baseline). Then
**diff-critic** on the branch diff + **@codex**; resolve threads; squash-merge on green CI + owner go.
`finish-slice`.

**Codex watch (this family's recurring class):** an endpoint that RETURNS data must gate on the READ
key (here `register.read` ✓). Confirm the `None`-governing branch returns `published:false` not a 500;
confirm org-level (not per-process) is intentional (the named deferral). Expect ≥1 round.
