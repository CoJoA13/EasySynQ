# S-web-3 Implementation Plan — Document Authoring

> Spec: `docs/superpowers/specs/2026-06-07-web-track-s-web-3-design.md`. Branch: `feat/s-web-3-authoring`.
> Method (S-web-1/2 precedent): spec+plan → TDD per task (test first where logic exists) → per-task quality review →
> whole-branch `diff-critic` before PR. CI: all five jobs green. **No migration** (head stays `0044`).
> Scope is locked to the **author's half** (create → submit-review); approve/inbox/release are S-web-5.

## Build sequence (dependency order)

### Phase 0 — backend: the affordance layer (no migration, no new key)

**T0.1 — extract the effective-permissions computation into a shared helper.** Pull the loop in
`api/authz.py:457-488` (`granted_permission_keys` → per-key `gather_grants` → `authorize` at `_resource_for(scope)`)
into `services/authz/…compute_effective_permissions(session, user, org_id, scope_level, scope_id)` returning the
`[{key, effect, source}]` list. Re-point the existing admin `GET /users/{id}/effective-permissions` at it (behaviour
byte-identical — its tests are the regression backstop). *Test:* the existing authz tests stay green.

**T0.2 — `GET /api/v1/me/permissions`** in `api/auth.py` (next to `/me`): **auth-only** (`Depends(get_current_user)`),
query `scope_level` (default `SYSTEM`) + `scope_id`; returns `{scope:{level,selector}, permissions:[…]}` via T0.1.
**Not** latch-exempt. *Tests:* `tests/integration/test_me_permissions.py` — 401 unauth; an **ordinary non-admin**
user (a content role, e.g. Author) gets its own keys (proves it is **not** `user.read`-gated); `scope_level=DOC_CLASS
&scope_id=<level>` shifts the answer; rich shape `{key,effect,source}`. *Contract:* document in `openapi.yaml`
(reuse the `EffectivePermissions` schema).

**T0.3 — `capabilities` block on `GET /documents/{id}`** (`api/documents.py:600-609`). Add a helper
`document_capabilities(session, caller, doc)`: for each of `document.checkout/edit/manage_metadata/submit/read_draft`
run `gather_grants`+`authorize` against `_document_scope_by_id(doc)`; for `document.release` build the SoD-enriched
scope via `enrich_release_sod_scope` against the **latest Approved version** (no Approved → false) with
`sig_hook=True`; `document.obsolete` against the plain scope with `sig_hook=True`. Return
`capabilities:{checkout,edit,manage_metadata,submit,release,obsolete,read_draft}` on the **detail** response only
(never the list serializer). *Tests:* `tests/integration/test_documents.py` (or a new `test_capabilities.py`) — a
**scoped Author** (`checkout/edit/submit/read_draft`=T, `manage_metadata`=F, `release`=F); a **Process Owner**
(`manage_metadata`=T); the **version author** with a release grant (`release`=F — SoD); a **distinct release-granted
user** (`release`=T). Run-scoped/delta assertions; a scoped (non-SYSTEM-override) user is mandatory. *Contract:* add
the optional `capabilities` object to the `Document` schema in `openapi.yaml`.

Phase-0 gate: `/check-api` + `redocly lint` + `/check-migrations` (no-op confirm).

### Phase 1 — front-end foundation (infra + types + affordance hook)

**T1.1 — types** (`apps/web/src/lib/types.ts`): add `MePermissions` (`{scope, permissions:[{key,effect,source}]}`),
`DocumentCapabilities`, extend `Document` with optional `capabilities`; `WorkingDraft` (`{id, document_id,
checked_out_by, checked_out_at, source_version_id, lock_ttl_seconds}`); `InitUploadResult` (`{dedup, object_key,
upload_url|null}`); `CheckinResult` (`DocumentVersion` + `change_detected`); `DocumentCreate`, `ClauseMapping`,
`ClauseMappingCreate`; `ChangeSignificance = "MAJOR"|"MINOR"`. `import type`.

**T1.2 — `lib/hash.ts` + `lib/upload.ts`.** `sha256Hex(blob)` via `crypto.subtle.digest` → hex;
`putToPresigned(url, file, contentType)` = raw `fetch` PUT, **no bearer**, `Content-Type` only. *Tests:* unit —
`sha256Hex` against a known vector; `putToPresigned` (MSW `http.put`) asserts method/body/`Content-Type` and **no
`Authorization`**. (Stub `crypto.subtle` in `setup.ts` only if jsdom lacks webcrypto.)

**T1.3 — `usePermissions()`** (`app/shell/usePermissions.ts`): `useQuery` over `GET /me/permissions`, boot-cached;
exposes `can(key)`. A parameterized variant for `scope_level=DOC_CLASS&scope_id`. *Test:* `usePermissions.test.tsx` —
`can()` true/false from a mocked response.

**T1.4 — authoring mutation hooks** (`features/authoring/hooks.ts` or one file each) on **`useApi().send`**:
`useCreateDocument`, `useCheckout`, `useHeartbeat`, `useBreakLock`, `useInitUpload`, `useCheckin`, `useMapClause`,
`useUnmapClause`, `useSubmitReview`, `useStartRevision`. Standardize `onSuccess` invalidations (`["document",id]`,
`["document-versions",id]`, `["where-used",id]`, `["documents"]`).

**T1.5 — MSW fixtures/handlers** (`src/test/msw/handlers.ts`): `GET /me/permissions`; `POST /documents` (201 + a
Draft; a 409 `singleton_exists` variant); `checkout` (200 + WorkingDraft; 409 `lock_conflict`); `heartbeat`;
`break-lock`; `versions:init-upload` (`dedup:false`+url and `dedup:true`+null variants); the **presigned PUT**
(`http.put("https://minio.test/*", …200)`); `checkin` (201 + version+`change_detected`; 422 empty-reason;
422 bad-significance); `clause-mappings` POST (201; 409 dup; 422 framework) + DELETE (204); `submit-review`
(200 InReview; 422 no-clause); `start-revision` (200 UnderRevision); `GET /documents/{id}` extended with
`capabilities`. Full-UUID ids, ISO timestamps, in lockstep with `lib/types.ts`.

### Phase 2 — the shared authoring pieces

**T2.1 — `CheckInPanel.tsx`** (`features/authoring/`): the doc-11 §5.4 card — lock banner (holder + TTL countdown +

**T2.2 — `ClauseMapper.tsx`** (`features/authoring/`): clause multi-select over `useClauses` (PDCA-banded, `★`;

### Phase 3 — the New-Document wizard

**T3.1 — `NewDocumentWizard.tsx`** (`features/authoring/`): a Mantine `<Stepper>` — (1) Metadata → `useCreateDocument`
(type picker from `useDocumentTypes`; re-check `document.create` at the chosen DOC_CLASS; validated `area_code`;
409 singleton inline); (2) Upload → `CheckInPanel` (significance defaults MAJOR); (3) Clauses → `ClauseMapper` (≥1 to
advance); (4) Review & submit → `useSubmitReview` → route to the Library with the new doc's drawer open. Step in the
URL; the doc id threads from step 1. *Tests:* the 4-step progression (each mutation gates the next via `waitFor`),
the singleton/422 surfaces, the create-at-step-1 copy; axe on each step.

**T3.2 — wire the entry point:** a **"New Document"** `<Button>` in the `LibraryPage` header (shown when
`usePermissions().can("document.create")`), opening the wizard (route `/library/new` or a focused modal — pick the
route for deep-linkability). *Test:* the button is hidden without the cap; shown with it.

### Phase 4 — "Author actions" in the drawer

**T4.1 — `AuthorActions.tsx`** (`features/authoring/`): the capability + state + lock-gated section added to the
S-web-2 `DocumentDrawer` (above the tabs). Draft/UnderRevision → `CheckInPanel` + `ClauseMapper` + Submit (enabled at
≥1 clause); Effective → Start revision; InReview → "Awaiting review" indicator; else read-only. Reads `capabilities`
from `useDocument(id)`. *Tests:* each state's affordance set; cap-gating (manage_metadata false hides the mapper);
start-revision on Effective; submit enabling on the clause count; the InReview indicator; axe with the section open.

**T4.2 — drawer integration:** mount `AuthorActions` in `DocumentDrawer.tsx` without disturbing the read-only tabs
(one-way dep `authoring → document`). Confirm no import cycle.

### Phase 5 — tests, a11y, gate

**T5.1** — fill any gaps so every screen/flow has component tests; **jest-axe `toHaveNoViolations`** on each wizard
step and the drawer-with-actions (release gate). Verify `crypto.subtle` in jsdom (stub in `setup.ts` if needed).
**T5.2** — full local gate: `cd apps/web && npm run lint && npm run typecheck && npm run build && npm test`;

### Phase 6 — review, live smoke, PR

**T6.1** — `diff-critic` subagent on the whole branch diff. Emphasis lenses: the **`capabilities` authz correctness**
(scoped user, version-relative SoD on `release`), `/me/permissions` **not** leaking beyond self / not `user.read`-
gated, the **presigned PUT carries no bearer**, run-scoped integration assertions, and DP-6 (no dead buttons).
**T6.2 — live smoke** against the running stack (`http://localhost`, WSL). **Operator precondition first** (run it
yourself in WSL): grant the demo user the authoring keys — the brief's `grant-role … "QMS Owner"` is **reads-only**;
instead grant **SYSTEM overrides** for `document.create/checkout/edit/manage_metadata/submit/read/read_draft`
(establish the exact CLI/admin-override command in this task and, if it needs documenting, add it to
`.claude/rules/dev-workflow.md`), then restart the `api` to refresh its permission cache. Smoke: create a doc via the
wizard → upload a file (real browser→MinIO PUT — **verify MinIO CORS** allows the cross-origin PUT; if not, configure
it in the compose/minio-init as part of this task) → map a clause → submit → see InReview; reopen via the drawer and
confirm the Author actions gate correctly.
**T6.3** — `/pr` → PR against protected `main`; all five CI jobs green → squash-merge.

## Risks / watch-items

- **`capabilities` authz correctness** is the load-bearing backend change — the integration test MUST use a **scoped**
  user (not the SYSTEM-override admin, who would see every cap true) and MUST prove the **version-author `release`=F**
  SoD case. Reuse `gather_grants`+`authorize`+`enrich_release_sod_scope`; don't hand-roll the SoD.
- **The presigned PUT** is the one cross-origin, no-bearer call — (a) `lib/upload.ts` must not attach `Authorization`;
  (b) **MinIO CORS** for a browser PUT is a *runtime* prerequisite (the only existing browser→MinIO path is a
  presigned GET via `window.open`, which needs no CORS) — verify/configure in T6.2 before declaring the smoke green.
- **`crypto.subtle` in jsdom** — confirm Node webcrypto is exposed in the vitest env; stub in `setup.ts` if absent.
- **Orphan Drafts** — abandoning the wizard after step 1 leaves a versionless Draft (no T8 discard in v1). Keep
  step-1 creation explicit + copy clear; documented limitation, not a bug.
- **The demo persona gap** — the seeded **Author** role can't map clauses (no `document.manage_metadata`); use a
  SYSTEM override bundle (or Process Owner + a matching process scope) so one user can complete create→submit.
- **Import cycles** (S-web-1 lesson) — `features/authoring/ → features/document/` only; the drawer imports
  `AuthorActions`, not vice-versa; the shell never imports features.
- **MSW `onUnhandledRequest:"error"`** — every authoring endpoint **and the cross-origin MinIO PUT** need default
  handlers, or tests fail.
- **Response/contract drift** — `/me/permissions` + the `capabilities` field must land in `openapi.yaml` (contracts
  CI is redocly-lint) in the same PR; reuse the existing `EffectivePermissions` schema.
